Known Issues

    

AFF Images

•Due to the nature of the format, large AFF images (>50GB) may take several minutes to analyze and mount.
•Write performance is quite poor for AFF images. If performance is a concern, consider mounting as a RAM drive and saving the modified image.

NTFS Images

•When mounting an NTFS image as read-only, you might get the error "X:\ is not accessible. The media is write protected". This error may be caused by the NTFS Access control list (ACL) preventing access to the drive. As a workaround, you can mount the image as read/write, allowing Windows to automatically fix the NTFS ACL. After the initial mount as a R/W device, you can then immediately dismount the drive and re-mount it as read only.

EnCase/SMART EWF Images

•When mounting an EnCase image as read/write, all writes to the image are stored in a delta file (.d01) with the same base name, leaving the original image file unmodified. When re-mounting, the delta file must be located in the same directory in order for modified image to be properly mounted.
