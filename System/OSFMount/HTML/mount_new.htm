Mounting a Virtual Drive

    

 

 

Source

Specify the source of the virtual disk.

 

Image file

The virtual disk will be backed by the specified image file on disk.

 

Image file in RAM

The specified image file will be loaded into RAM, then mounted as a virtual disk.

 

Empty RAM drive

The virtual disk will be backed by an empty RAM drive. The user may be prompted to format the drive.

 

 

Image file

Specify the image filepath on disk.

 

•For Split Raw Images, specify the first file (.000 or .001 file extension). The file extensions must be in the .0XX format where the first file must have .000 or .001 as the file extension, and the subsequent files with incrementally higher extensions (ie. .002, .003, .004, etc).
•For AFD Images, specify the directory path (.AFD extension). The directory must have the .AFD extension and the contained files must have filenames in the format file_00n.AFF where n is an integer starting from 0.
•For split EnCase/SMART Images, specify the first file (.E01/.S01 file extension) in the set. The remaining split files (eg. image.E02, image.E03) and delta files (eg. image.d01) must be in the same directory.

Volume options

 

Mount specified partition

Mount the partition specified by the image file offset and drive size.

 

Select... (Select partition)

If the image file selected contains a partition table (MBR or GPT) with more than 1 partition, then OSFMount will display a window to allow a particular partition to be selected. Based on the user selection, The byte offset and partition size will be automatically calculated.

"Use entire image file" will result in the entire image file being mounted as a drive. In many cases, Windows will prompt the user to format the drive since Windows expects a boot sector (e.g. NTFS) for the partition but instead reads the Master Boot Record (which does not correspond to any particular file system's boot sector). It is recommended that the "Use entire image file" option is not used for forensic analysis of an image.

 

Image file offset

The start offset in the image to mount. This field provides support for cases where the image file contains more than one partition (i.e. An image of the entire physical device, and not just a partition). This specifies the offset for the start of the partition (i.e. the starting byte for the boot sector for the partition).

 

Drive size

The size of the drive. For image files, this should be unaltered as the size shall be automatically determined. For empty RAM drives, the size of the drive must be specified.

 

Mount all partitions

Mount all partitions in the image file as a separate drive. The offset and size of each drive is automatically determined by the entries in the partition table.

 

 

Mount options

 

Drive letter

Specify the drive letter to mount the image file as. By default, the next available drive letter is chosen.

 

Type of drive

Force the virtual disk to be mounted as a certain drive type. For forensics analysis, this field should be unaltered as the device type shall be automatically determined.

 

Read-only drive

Specify the virtual disk to be read-only. When this option is selected, the contents of the original disk image file is not changed and as such, is the recommended option for forensic analysis. This is the default behavior.

 

Removable media

The drive is created with removable and hot-plug characteristics. This changes the caching behavior of the mounted disk.

 

 

OK

Mount the virtual drive with the selected settings.

 

 

Cancel

Cancel mounting the drive.
